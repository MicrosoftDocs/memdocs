# Offline feedback submission fails when using UploadOfflineFeedback.exe

#### Prerequisites
To apply this hotfix, you must be using Configuration Manager, versions 2409 (with Update rollup 30385346 ) versions 2503 (with Update rollup 32851084 installed) and 2509.

[2409:Update rollup for Microsoft Configuration Manager version 2409](../../hotfix/2409/30385346.md)
[2503: Update rollup for Microsoft Configuration Manager version 2503](../../hotfix/2503/32851084.md)

## Summary KB36419222
When you submit offline product feedback from the Configuration Manager console by using the **UploadOfflineFeedback.exe** command-line tool, the submission may fail with an unexpected error and the feedback is not uploaded to Microsoft.

## Issue details
This issue occurs due to a dependency version mismatch for **Microsoft.Identity.Client**, which prevents the offline feedback client from authenticating and completing the submission process.

## Symptoms
When you run **UploadOfflineFeedback.exe** from a command prompt to submit a previously saved offline feedback package, the command fails and returns an error similar to the following:



```text
UploadOfflineFeedback Error: 2 : An unexpected error 'System.IO.FileLoadException' occurred.
Could not load file or assembly 'Microsoft.Identity.Client, Version=4.61.3.0, Culture=neutral,
PublicKeyToken=0a613f4dd989e8ae' or one of its dependencies.
